AWS⇄GCP Data Engineering

Category

Key Differences / Interview Notes
Object & File Storage
Object Storage
Data lake foundation
S3
Simple Storage Service. Buckets, prefixes, versioning, lifecycle policies
★ Cornerstone
Cloud Storage (GCS)
Buckets, object versioning, storage classes
★ Cornerstone
Both support multipart upload, ACLs, signed URLs. GCS has strong consistency by default.
Data Lake Format
S3 + Lake Formation
Governed tables, row/col security
GCS + Dataplex
Unified data management, auto discovery
Dataplex adds governance layer on top of GCS/BQ.
Batch Processing
Managed Spark/Hadoop
ETL backbone
EMR
Elastic MapReduce. Managed Spark, Hive clusters.
MANAGED
Dataproc
Managed Spark/Hadoop. Fast spin-up (~90s).
MANAGED
Dataproc clusters start faster. Both support autoscaling.
Serverless ETL
AWS Glue
Serverless Spark ETL. Job bookmarks, catalog.
SERVERLESS
Dataflow
Managed Apache Beam. Batch + streaming unified.
SERVERLESS
Glue = Spark-based; Dataflow = Beam-based.
Data Warehouse
Cloud Data Warehouse
Core analytics store
Redshift
MPP columnar DW. RA3 nodes. Redshift Serverless.
★ Cornerstone
BigQuery
Serverless, fully managed DW. Dremel engine. BQML built-in.
★ Cornerstone
BigQuery is serverless by design. Redshift requires cluster decisions.
Orchestration
Managed Airflow
Pipeline scheduling
MWAA
Managed Workflows for Apache Airflow.
Cloud Composer
Managed Airflow on GKE.
Both are managed Airflow — same DAG code.
★ Cornerstone Core service
MANAGED Cluster-based
SERVERLESS No-infra
